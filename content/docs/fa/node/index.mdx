## بررسی اجمالی

نود PasarGuard توسط تیم PasarGuard توسعه داده شده است تا راه‌حلی پایدار، مقیاس‌پذیر و کارآمد برای مدیریت هسته‌های پروکسی فراهم کند. این نود برای کار یکپارچه با پنل PasarGuard یا به عنوان یک راه‌حل مستقل طراحی شده است.

### چرا از نود PasarGuard استفاده کنیم؟

نود PasarGuard برای انعطاف‌پذیری و جهانی بودن طراحی شده است:

- **طراحی جهانی**: می‌تواند در هر پروژه‌ای استفاده شود، حتی بدون پنل PasarGuard
- **یکپارچه‌سازی ساده**: اجرای نودها با یک اسکریپت ساده با استفاده از کتابخانه‌های رسمی
- **پشتیبانی چند هسته‌ای**: برنامه‌ریزی برای گسترش پشتیبانی از هسته‌های مختلف پس از مرحله تست
- **معماری مقیاس‌پذیر**: ساخته شده برای عملکرد و قابلیت اطمینان

### هسته‌های پشتیبانی شده

| هسته | پشتیبانی |
| ---- | ------- |
| [Xray-core](https://github.com/XTLS/Xray-core) | ✅ پشتیبانی می‌شود |
| [Sing-box](https://github.com/SagerNet/sing-box) | ⏳ به زودی |
| [V2ray-core](https://github.com/v2fly/v2ray-core) | ⏳ به زودی |

### ویژگی‌های کلیدی

- **پشتیبانی از REST API و gRPC**: پروتکل ارتباطی مورد علاقه خود را انتخاب کنید
- **آمار لحظه‌ای**: نظارت بر آمار سیستم، بکند و کاربران
- **مدیریت کاربر**: همگام‌سازی و مدیریت پویای کاربران
- **لاگ زنده**: پخش لاگ‌ها در زمان واقعی
- **کنترل ترافیک**: نظارت و کنترل ترافیک به ازای هر کاربر
- **پشتیبانی SSL/TLS**: تولید و مدیریت گواهی داخلی

## شروع سریع

با نصب یک کلیکی ما با نود PasarGuard شروع کنید:

```bash
sudo bash -c "$(curl -sL https://github.com/PasarGuard/scripts/raw/main/pg-node.sh)" @ install
```

## معماری

نود PasarGuard با این ابزارها ساخته شده است:

- **Go**: بکند با کارایی بالا
- **gRPC & REST API**: پشتیبانی از دو پروتکل
- **Protobuf**: سریال‌سازی کارآمد داده
- **Docker**: استقرار آماده کانتینر
