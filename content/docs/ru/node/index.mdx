## Обзор

PasarGuard Node разработан командой PasarGuard для обеспечения стабильного, масштабируемого и эффективного решения для управления прокси-ядрами. Он предназначен для бесшовной работы с панелью PasarGuard или в качестве автономного решения.

### Зачем использовать PasarGuard Node?

PasarGuard Node разработан с учетом гибкости и универсальности:

- **Универсальный дизайн**: Может использоваться в любом проекте, даже без панели PasarGuard
- **Простая интеграция**: Запускайте ноды с помощью простого скрипта, используя официальные библиотеки
- **Поддержка нескольких ядер**: Планируется расширение для поддержки различных ядер после этапа тестирования
- **Масштабируемая архитектура**: Создан для производительности и надежности

### Поддерживаемые ядра

| Ядро | Поддержка |
| ---- | ------- |
| [Xray-core](https://github.com/XTLS/Xray-core) | ✅ Поддерживается |
| [Sing-box](https://github.com/SagerNet/sing-box) | ⏳ Скоро |
| [V2ray-core](https://github.com/v2fly/v2ray-core) | ⏳ Скоро |

### Основные возможности

- **Поддержка gRPC и REST API**: Выберите предпочитаемый протокол связи
- **Статистика в реальном времени**: Мониторинг системы, бэкенда и статистики пользователей
- **Управление пользователями**: Динамическая синхронизация и управление пользователями
- **Живые логи**: Потоковая передача логов в реальном времени
- **Контроль трафика**: Мониторинг и контроль трафика для каждого пользователя
- **Поддержка SSL/TLS**: Встроенная генерация и управление сертификатами

## Архитектура

Нод PasarGuard построен с использованием:

- **Go**: Высокопроизводительный бэкенд
- **gRPC & REST API**: Поддержка двух протоколов
- **Protobuf**: Эффективная сериализация данных
- **Docker**: Готовое к контейнерному развертыванию
